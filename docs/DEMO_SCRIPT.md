# SignRepair Demo Script

Five-minute demo path for mocked or low-risk browser demo.

## Opening lines

- "This is not an interpreter."
- "The product is honest uncertainty."
- "The system stores landmark-derived data, not video."
- "Deaf-led review is required before real deployment."

## Demo path

1. Open Home.
   Explain: `SignRepair` and `Privacy-first sign evidence and repair prototype.`
   Point to safety box and consent copy.
2. Start live demo with mocked or demo webcam flow.
   Explain that built-in candidates are illustrative known-candidate demo labels only.
3. Trigger uncertain candidate state.
   Say that product does not hide uncertainty.
4. Show Translation Debt.
   Explain which evidence is missing or weak.
5. Try Cue Patch.
   Show smallest useful recapture instead of default full repeat.
6. Open Verify with `sample clip.mp4`.
   Show exact model output, expected human reference, and match result without claiming translation.
7. Open Motion Replay Receipt.
   Explain that it is landmark-only inspectability, not proof of meaning.
8. Show SignForm Ledger.
   Explain that these are coarse sign-form evidence slots, not official ASL analysis.
9. Save local repair or Confusion Twin memory.
   Explain that local repair memory is optional and stays on device.
10. Open Minimal Pair Lab.
   Compare confusing local candidates with landmark-derived examples only.
11. Open Evidence Health.
    Show healthy, watch, needs-review, and unknown as local evidence quality only.
12. Export local data.
    Show that export contains landmark-derived data and no raw-video-like fields.
13. Clear all local data.
    Confirm local memory can be removed from device.

## Closing lines

- "This is not an interpreter."
- "The product is honest uncertainty."
- "The system stores landmark-derived data, not video."
- "Deaf-led review is required before real deployment."
